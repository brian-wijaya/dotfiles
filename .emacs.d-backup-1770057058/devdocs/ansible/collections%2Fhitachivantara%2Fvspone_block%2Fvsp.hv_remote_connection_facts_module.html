hitachivantara.vspone_block.vsp.hv_remote_connection_facts module – Retrieves Remote connection details from Hitachi VSP storage systems.

Note

This module is part of the hitachivantara.vspone_block collection (version 3.5.1).

You might already have this collection installed if you are using the ansible package. It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install hitachivantara.vspone_block. You need further requirements to be able to use this module, see Requirements for details.

To use it in a playbook, specify: hitachivantara.vspone_block.vsp.hv_remote_connection_facts.

New in hitachivantara.vspone_block 3.3.0

Synopsis

    • This module retrieves information about remote connections from Hitachi VSP storage systems. - For examples go to URL

    https://github.com/hitachi-vantara/vspone-block-ansible/blob/main/playbooks/vsp_direct/remote_connection_facts.yml

Requirements

The below requirements are needed on the host that executes this module.

  • python >= 3.9

Examples

- name: Get all remote connection details
  hitachivantara.vspone_block.vsp.hv_remote_connection_facts:
    connection_info:
      address: storage1.company.com
      username: "admin"
      password: "password"

- name: Get remote connection details using path group id
  hitachivantara.vspone_block.vsp.hv_remote_connection:
    connection_info:
      address: storage1.company.com
      username: "admin"
      password: "password"
    spec:
      path_group_id: 101

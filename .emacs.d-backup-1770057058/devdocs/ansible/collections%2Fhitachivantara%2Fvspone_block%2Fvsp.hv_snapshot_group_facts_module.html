hitachivantara.vspone_block.vsp.hv_snapshot_group_facts module – Retrieves snapshot information in units of snapshot groups from Hitachi VSP storage systems.

Note

This module is part of the hitachivantara.vspone_block collection (version 3.5.1).

You might already have this collection installed if you are using the ansible package. It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install hitachivantara.vspone_block. You need further requirements to be able to use this module, see Requirements for details.

To use it in a playbook, specify: hitachivantara.vspone_block.vsp.hv_snapshot_group_facts.

New in hitachivantara.vspone_block 3.2.0

Synopsis

  • This module retrieves information about snapshots in units of snapshot groups from Hitachi VSP storage systems.
  • For examples go to URL https://github.com/hitachi-vantara/vspone-block-ansible/blob/main/playbooks/vsp_direct/snapshot_group_facts.yml

Requirements

The below requirements are needed on the host that executes this module.

  • python >= 3.9

Examples

- name: Gather snapshot facts with primary volume and mirror unit ID
  hitachivantara.vspone_block.vsp.hv_snapshot_group_facts:
    connection_info:
      address: storage1.company.com
      username: "admin"
      password: "secret"
    spec:
      snapshot_group_name: 'NewNameSPG'

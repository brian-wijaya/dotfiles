CLI API

Bun and esbuild both provide a command-line interface. 
# esbuild
esbuild <entrypoint> --outdir=out --bundle

# bun

In Bunâ€™s CLI, simple boolean flags like --minify do not accept an argument. Other flags like --outdir <path> do accept an argument; these flags can be written as --outdir out or --outdir=out. Some flags like --define can be specified several times: --define foo=bar --define bar=baz.

Plugin API

Bunâ€™s plugin API is designed to be esbuild compatible. Bun doesnâ€™t support esbuildâ€™s entire plugin API surface, but the core functionality is implemented. Many third-party esbuild plugins will work out of the box with Bun.
Long term, we aim for feature parity with esbuildâ€™s API, so if something doesnâ€™t work please file an issue to help us prioritize.
Plugins in Bun and esbuild are defined with a builder object. 
import type { BunPlugin } from "bun";

const myPlugin: BunPlugin = {
  name: "my-plugin",
  setup(builder) {
    // define plugin
  },
};
The builder object provides some methods for hooking into parts of the bundling process. Bun implements onResolve and onLoad; it does not yet implement the esbuild hooks onStart, onEnd, and onDispose, and resolve utilities. initialOptions is partially implemented, being read-only and only having a subset of esbuildâ€™s options; use config (same thing but with Bunâ€™s BuildConfig format) instead. 
import type { BunPlugin } from "bun";
const myPlugin: BunPlugin = {
  name: "my-plugin",
  setup(builder) {
    builder.onResolve(
      {
        /* onResolve.options */
      },
      args => {
        return {
          /* onResolve.results */
        };
      },
    );
    builder.onLoad(
      {
        /* onLoad.options */
      },
      args => {
        return {
          /* onLoad.results */
        };
      },
    );
  },
};

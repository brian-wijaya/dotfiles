Failure Modes Have Names
KNOW THESE BY NAME:

Race condition
  Two writes at same time, last one wins
  Fix: mutex, atomic, channel

Deadlock
  A waits for B, B waits for A
  Fix: lock ordering, timeouts

Livelock
  Both keep yielding, no progress
  Fix: randomized backoff

Memory leak
  Allocated but never freed
  Fix: GC, RAII, ownership

Use-after-free
  Access memory after freeing
  Fix: Rust borrow checker

Buffer overflow
  Write past array bounds
  Fix: bounds checking, safe langs

SQL injection
  '; DROP TABLE users;--
  Fix: parameterized queries

Thundering herd
  Cache expires, all clients hit DB
  Fix: jitter, stampede protection

Cascading failure
  Service A fails â†’ B fails â†’ C fails
  Fix: circuit breakers, bulkheads

Split brain
  Network partition, both think leader
